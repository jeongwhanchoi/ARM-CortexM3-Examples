
	ADC2 Example 설명

	source_code -> Example -> 5_ADC1 -> ADC2(Polling)  - 예제 경로

-------------------------------------------------------------------------------

	ADC2 예제는 ADC를 사용하여 Potentio(가변저항)의 ADC값을 Polling하여 읽어오는 예제 입니다.
	
	Polling은 원하는 곳을 계속적으로 확인을 하는 방법입니다. Polling은 인지하지 못했을 뿐 많은 방식으로 이용됩니다.

	예를들어 

	while(1)
	{ 
		if(a == 1)	b = 2 
	}
	
	이러한 간단한 코드도 Polling 기능이 사용됩니다. 지속적으로 a라는 변수를 관찰하다가 1이 되었을때 b라는 변수를
	
	2로 바꾸게됩니다. a라는 변수를 While이 실행될 때마다 계속 확인하기 때문에 Polling이 되는 것입니다. Polling의 단점은
	
	정확한 값을 못받아 올 수도 있습니다. 왜냐하면 만약 while문의 코드가 복잡하여 1Cycle을 도는데 1초가 걸린다면 a변수는 
	
	1초마다 확인될 것인데, a변수가 바뀌는 사이클이 1초보다 짧다면 한번 확인하고 되돌아오기도 전에 값이 몇번이나 바뀔수가 있기 때문입니다.
	
	이러한 이유 때문에 Interrupt를 사용합니다. 정확한 이해는 이 예제코드를 살펴보면 됩니다. 기본적인 ADC 설정은 
	
	이전 ADC1 예제와 동일하며 따로 Interrupt를 설정하지 않고,  While문에서 100ms 마다 adc_value의 값을 읽어 옵니다. 
	
	adc_value 값은 NXP Board의 Potentio(가변저항)을 돌리면 ADC 값이 바뀌게 되는데 Polling하는 시간이 길다면 
	
	가변저항을 빠르게 돌릴 경우 값을 다 감지 하지 못하게 됩니다. 바뀌는 순간을 체크하는게 아니라 내가 원하는 시간 마다 

	체크를하기 때문입니다. 
	
	ADC1 예제와 유사하기 내용이기때문에 설정에 대해서는 ADC1 예제를 참고하여 주시고, Interrupt와 Polling의 차이점을 
	
	숙지하시기 바랍니다.
